## Verdict: APPROVE

## Summary
Moves the `setPage(0)` call from the render phase to a `useEffect` with `filteredData`, `page`, and `rowsPerPage` dependencies. Also extracts `filteredData` into a `useMemo` so both the effect and the render path use the same stable reference. This eliminates a React anti-pattern (render-phase state update causing an extra synchronous render cycle).

## Findings

### Positive
- Memoising `filteredData` as a prerequisite for the effect is a thoughtful design — avoids a potential double-trigger where both the render and the effect recompute the filtered list independently.
