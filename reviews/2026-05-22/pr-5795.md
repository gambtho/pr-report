## Verdict: APPROVE

## Summary
Drops the complex ESRP-based `notarize` and `stapler` CI jobs (Windows runner, Azure login, NuGet ESRP client) and replaces them with electron-builder's native notarization in the `build-mac` step using standard Apple credentials (APPLEID, APPLEIDPASS, APPLETEAMID). Enables `hardenedRuntime` and removes the `identity: "-"` override that was disabling code signing. The `verify-notarization` job is retained. The PR author (a maintainer) tested the workflow branch successfully.

## Findings

### Warnings
- **[MEDIUM]** `.github/workflows/app-artifacts-mac.yml:51` — Three new secret names are used (`APPLEID`, `APPLEIDPASS`, `APPLETEAMID`) which differ from the previous `APPLE_TEAM_ID`. Ensure these secrets are configured in the repo/org settings before merging, or the signed build step will silently use empty values and produce an unsigned artifact.

### Positive
- Removes ~93 lines of Windows-runner ESRP infrastructure — significant reduction in CI complexity and attack surface.
