## Verdict: NEEDS_DISCUSSION

## Summary
Introduces the `--oidc-auto-login` flag (backend, Helm, frontend) that automatically redirects unauthenticated users to the OIDC provider. The updated PR now correctly handles sessionStorage lifecycle: the flag is cleared on successful OIDC callback and on explicit 'Sign In' clicks, preventing redirect loops. A bonus fix in `ThemeProviderNexti18n` guards against empty locale strings. The implementation is solid but still lacks a frontend unit test for the auto-login useEffect path.

## Findings

### Warnings
- **[MEDIUM]** `frontend/src/components/authchooser/index.tsx:82` — No Vitest test covers the auto-login `useEffect` path in `AuthChooser`. This is a critical flow (automatic redirect) — missing test coverage means regressions won't be caught. A test should mock `sessionStorage`, `window.location`, and the `oidcAutoLogin` selector.

### Suggestions
- **[LOW]** `frontend/src/components/authchooser/index.tsx:95` — `window.location.href = oidcUrl.toString()` performs a hard navigation that bypasses React Router. This is intentional for the full-page OIDC redirect, but a comment explaining the choice would aid future readers.
- **[LOW]** `frontend/src/components/oidcauth/index.tsx:28` — `new URL(window.location.href).searchParams.get('cluster')` is used via `useLocation` — the `cluster` variable is already validated non-null before the `if (!window.opener)` branch, which is good. Consider adding a JSDoc comment explaining the two-flow distinction (popup vs. full-page redirect).

### Positive
- The previous sessionStorage edge case (flag not cleared after successful OIDC auth) is now fixed: `sessionStorage.removeItem` is called in both `oidcauth/index.tsx` (successful callback) and `authchooser/index.tsx` (manual sign-in click).
- Bonus fix in `ThemeProviderNexti18n.tsx` guards `getLocale('')` with an early return, preventing a crash when `i18n.language` is falsy.
