## Verdict: REQUEST_CHANGES

## Summary
Extends the Helm JSON schema to allow `valueFrom.configMapKeyRef` and `valueFrom.secretKeyRef` in the `env` array, replacing the previous `required: ["name", "value"]` constraint with a `oneOf` that enforces either `value` or `valueFrom`. This is a useful enhancement but still has two outstanding issues from the initial review.

## Findings

### Warnings
- **[HIGH]** `charts/headlamp/values.schema.json` — `valueFrom` only covers `configMapKeyRef` and `secretKeyRef`. Kubernetes `EnvVarSource` also supports `fieldRef` (Downward API — Pod metadata) and `resourceFieldRef` (container resource limits/requests). Omitting them means valid Kubernetes env-var configs will fail Helm schema validation. Add both `fieldRef` and `resourceFieldRef` as additional `oneOf` alternatives, or remove `additionalProperties: false` from the `valueFrom` object.
- **[MEDIUM]** `charts/headlamp/values.schema.json` — Commit title `feat(helm): Add valueFrom support for environment variables in chart` uses Conventional Commits style. Project convention is `helm: deployment: Add valueFrom support for env vars`. This will likely trigger a CHANGES_REQUESTED from the maintainer before merge.
