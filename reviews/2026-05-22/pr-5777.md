## Verdict: APPROVE

## Summary
Fixes a cache invalidation gap where only the namespace-specific key was deleted on informer events, leaving stale all-namespace aggregated data. The fix delegates to `DeleteKeys(key, k8scache)` which strips the namespace segment and also deletes the all-namespace variant. A new test `TestRunInformerToWatchDeletesAllNamespaceKey` pre-populates both keys and asserts both are gone after the informer fires.

## Findings

### Suggestions
- **[LOW]** `backend/pkg/k8cache/cacheInvalidation.go` — Log message 'and any namespace-stripped variant will be deleted from the cache' is grammatically awkward; consider 'will delete both the namespaced key and the all-namespace variant'.

### Positive
- Test uses `assert.Eventually` with a generous 2 s timeout — robust against scheduler jitter without being brittle.
