## Verdict: APPROVE

## Summary
Fixes a bug where `oidcCACert` defaulting to `""` resulted in a non-nil pointer to an empty string being passed to `ConfigureTLSContext`, which then attempted to parse an empty PEM and returned early, breaking OIDC for any public provider (Entra ID, Okta, etc.) without a custom CA. The fix applies in two layers: `auth.go` guards the PEM parsing with `*caCert != ""`, and `kubeconfig.go` only sets `CACert` to a non-nil pointer when the cert string is non-empty. A targeted test verifies context is unchanged for the empty-cert case.

## Findings

### Positive
- Defence-in-depth: the fix is applied both at the point of pointer creation (`kubeconfig.go`) and at the point of use (`auth.go`), so neither layer alone is a single point of failure.
