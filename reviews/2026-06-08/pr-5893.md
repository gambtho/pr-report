## Summary
Purely additive PR that creates `DeleteButton.stories.tsx` with five Storybook stories (Default, WithAfterConfirmCallback, MenuButtonStyle, PodEvict, NoItem) plus the corresponding storyshot snapshots. Fills a gap in visual test coverage for a component that previously had no stories. The mock item pattern follows `ScaleMultipleButton.stories.tsx`.

## Findings

### Suggestions
- **[LOW]** `frontend/src/components/common/Resource/DeleteButton.stories.tsx:88` — The `PodEvict` story description says "The button shows 'Evict' instead of 'Delete' when the useEvict cluster setting is enabled," but the snapshot confirms the button renders as "Evict" for a Pod item without explicitly enabling any cluster setting. The description is slightly misleading about the triggering condition; consider clarifying to "When the item is a Pod, the button renders as 'Evict' by default" to match observed behavior.

### Positive
- Five stories cover the main visual states cleanly: icon button, menu-list item, pod-specific evict path, and no-item (null-render) — comprehensive coverage for a simple component.
- Mock item construction follows the established `as unknown as KubeObject` pattern from `ScaleMultipleButton.stories.tsx`, keeping things consistent without introducing new testing patterns.
