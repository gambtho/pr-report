## Verdict: APPROVE

## Summary
The stale `useMemo(() => getTablesRowsPerPage(...), [])` anti-pattern has been correctly replaced with a direct call, and a `useEffect` now synchronises `rowsPerPage` with the new default whenever settings change — but only when no URL param is pinning it. The fix for the URL-param guard is careful: it checks `location.search` for the actual key name (accounting for the `prefix` separator), so navigating back to a page that had a URL-stored page size doesn't wipe it out. All three commit subjects include the correct `frontend: SimpleTable:` prefix.

## Findings

### Suggestions
- **[LOW]** `frontend/src/components/common/SimpleTable.tsx` — Three separate commits for a 14-line change may warrant squashing into one before merge, but this is cosmetic and up to the maintainer's preference.

### Positive
- The `!hasURLParam` guard in the `useEffect` correctly avoids overriding a URL-pinned `perPage` value when the user settings change — the right behaviour.
- `getTablesRowsPerPage(rowsPerPageOptions[0])` is now a plain render-time call; removing the misleading `// eslint-disable-next-line react-hooks/exhaustive-deps` suppression cleans up the file.
