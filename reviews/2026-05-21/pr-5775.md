## Verdict: APPROVE

## Summary
Fixes a `window` event listener accumulation in the Backstage embedding path: `setupBackstageMessageReceiver()` now returns a cleanup function, and the `useEffect` in `Home/index.tsx` returns it as the cleanup callback. The root `package-lock.json` lockfile removes several unused dev dependencies (`@epic-web/invariant`, `cross-env`, `cross-spawn`, `isexe`, `path-key`, `shebang-command`, `shebang-regex`, `which`) which were likely left over from a prior tooling change. Author confirms `npm run frontend:test`, `tsc`, and `lint` all pass.

## Findings

### Positive
- The fix correctly wires up a `useEffect` cleanup via the standard React pattern (`return () => window.removeEventListener(...)`). The `if (isBackstage()) { return setupBackstageMessageReceiver(); }` guard correctly propagates `undefined` as the cleanup when not running in Backstage context, which is valid React behavior. The lockfile pruning removes dead weight without touching any source files.
