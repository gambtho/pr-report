## Verdict: NEEDS_DISCUSSION

## Summary
Adds a `--oidc-auto-login` CLI flag (and `config.oidc.autoLogin` Helm value) that, when enabled, automatically redirects users arriving without an active session on an OIDC cluster directly to the IdP, bypassing the "Sign In" button. The implementation covers the full stack: backend config, `/config` JSON response, Redux `configSlice`, `AuthChooser` redirect logic, `OIDCAuth` full-page-flow handling, a `ThemeProviderNexti18n` race-condition fix, and Helm chart templates plus a chart test. Author notes that backend tests were not run locally, and that the feature has been deployed to production at their company.

## Findings

### Warnings
- **[MEDIUM]** `frontend/src/components/authchooser/index.tsx:94` — `sessionStorage.getItem('oidc_auto_login_attempted')` is cleared only in the `OIDCAuth` callback for full-page flow (after a successful OIDC round-trip). If a user logs out explicitly via the Headlamp logout action (or the session is invalidated server-side without going through the callback), the flag remains set and auto-login will not re-trigger until the tab/session is closed. This could leave the user stuck on the login chooser without the auto-redirect they expect. Consider clearing the flag on explicit logout or when the user arrives at `AuthChooser` in a logged-out state.
- **[LOW]** `frontend/src/components/authchooser/index.tsx:97` — `oidcUrl.searchParams.set('dt', Date())` appends a human-readable date string (e.g. `"Thu May 21 2026 07:37:28 GMT+0000"`) to the OIDC redirect URL. This parameter is not standard for OIDC, its intent is unclear (cache-busting? loop detection?), and some strict OIDC providers may reject unknown query parameters. A random nonce or simple unix timestamp (e.g. `Date.now()`) would be cleaner if the goal is uniqueness; or it can be removed if not required.

### Suggestions
- **[MEDIUM]** `frontend/src/components/authchooser/index.tsx` — No frontend test covers the auto-redirect behavior. The backend test `TestGetConfigIncludesOidcAutoLogin` is present, but there is no test verifying that the `oidcAutoLogin === true` + `auth_type === 'oidc'` condition in `AuthChooser` triggers the redirect (and that the sessionStorage guard prevents an infinite loop). Given that this is security-adjacent behavior, a unit test with a mocked `window.location` would strengthen confidence.
- **[LOW]** `backend/cmd/headlamp.go:2158` — Author noted that backend Go tests were not run locally. CI should be confirmed green before merge, particularly for `TestGetConfigIncludesOidcAutoLogin`.

### Positive
- The `sessionStorage` loop-prevention mechanism is well-conceived and correctly prevents infinite redirects when the user cancels at the IdP.
- The `ThemeProviderNexti18n` race-condition fix (`getLocale` early-return on falsy locale; `useState(i18n.language || 'en')`) is a genuine defensive improvement that stands on its own merits regardless of the oidc-auto-login feature.
