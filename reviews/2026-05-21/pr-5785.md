## Verdict: APPROVE

## Summary
Replaces the deprecated `addListener`/`removeListener` API on `MediaQueryList` with the standard `addEventListener('change', …)` / `removeEventListener('change', …)` in `frontend/src/lib/themes.ts`. The handler type is narrowed from `MediaQueryListEvent | MediaQueryList` to `MediaQueryListEvent`, which is correct for the `change` event. All three mock usages in the test file are updated in parallel.

## Findings

### Suggestions
- **[LOW]** `frontend/src/lib/themes.ts:581` — The `// eslint-disable-next-line react-hooks/exhaustive-deps` comment above the empty `[]` dependency array is intentional (the effect should only run once on mount). A brief inline comment like `// intentional: register once on mount` would make it clearer to future readers that the suppression is deliberate and not a forgotten fix.

### Positive
- Clean modernisation of the deprecated `MediaQueryList` listener API. The type narrowing of the handler parameter from `MediaQueryListEvent | MediaQueryList` to `MediaQueryListEvent` is strictly more correct and removes the unreachable `MediaQueryList` union branch. Test coverage updated in lock-step with the implementation.
