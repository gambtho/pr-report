## Summary
Adds end-to-end support for operator-configurable external sidebar links via a new `externalLinks` Helm value, `HEADLAMP_CONFIG_EXTERNAL_LINKS` env var, `--external-links` CLI flag, and a corresponding Redux `configSlice` extension. The feature concept is sound, but the PR contains **unresolved Git merge conflict markers** in three backend files that prevent compilation.

## Findings

### Critical
- **[HIGH]** `backend/cmd/server.go:98` — **Unresolved merge conflict markers** (`<<<<<<< HEAD`, `=======`, `>>>>>>> feat: Add configurable external links...`) are present in `backend/cmd/server.go`, `backend/pkg/config/config.go`, and `backend/pkg/headlampconfig/headlampConfig.go`. The Go compiler will reject these files. The branch must be rebased/merged and conflicts resolved before this PR can be reviewed further.

### Warnings
- **[HIGH]** `backend/cmd/headlamp.go` — Commit title uses `feat:` Conventional Commits format (`feat: Add configurable external links to sidebar via Helm values`) instead of the project convention `area: Component: Description` (e.g. `backend/frontend/helm: Sidebar: Add configurable external links`). @illume typically requests a format correction before merging.

### Suggestions
- **[MEDIUM]** `backend/cmd/headlamp.go:2156` — External links JSON is parsed on every `/config` request. If parsing fails, the error is logged but the response silently returns an empty `externalLinks` array. Parsing at startup (in `buildHeadlampCFG`) and failing fast with a startup error would surface misconfiguration earlier.
- **[LOW]** `frontend/src/components/Sidebar/useSidebarItems.tsx:453` — The sidebar group icon (`'mdi:link-variant'`) and per-link fallback icon (`'mdi:link'`) are hardcoded. Since `ExternalLink` already has an optional `icon` field, consider allowing the group icon to be configurable in a future iteration, or at least document the hardcoded values.

### Positive
- The Redux slice, Storybook fixture, and unit test (`configSlice.test.ts`) are well-structured additions. The frontend correctly falls back to `|| []` and only renders the sidebar group when links are present.
