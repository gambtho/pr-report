## Summary

The implementation correctly adopts `addEventListener('change', …)` / `removeEventListener('change', …)` as the primary path, with a legacy `addListener`/`removeListener` fallback for older Safari/WebViews. Tests have been updated to mock the new API and verify handler identity. The remaining blockers are the missing SubArea in the commit subject and a potential ESLint lint failure from the legacy fallback lines.
