## Verdict: REQUEST_CHANGES

## Summary

The feature is end-to-end and well-structured (Helm → env → backend flag → `/config` API → Redux → sidebar). Backend tests for `getConfig` cover the new `ExternalLinks` field. However, the commit format is still using Conventional Commits (`feat:`) instead of the project convention, a URL-safety bug in `safeLinkUrl` lowercases the URL itself (losing case-sensitive path components), and @illume reports backend tests failing in CI.

## Findings

### Warnings
- **[HIGH]** `commit: "feat: add configurable external links to sidebar via Helm values"` — Conventional Commits format; @illume explicitly flagged this. Correct format: `frontend: Sidebar: Add configurable external links via Helm values` (or split into `frontend:` and `backend:` commits if preferred). Verb must also be capitalised.
- **[HIGH]** `frontend/src/components/Sidebar/useSidebarItems.tsx:55` — `safeLinkUrl` returns `url.toLowerCase()` instead of the original `url`. This silently lowercases URL paths (e.g. `https://example.com/MyPath` → `https://example.com/mypath`), breaking case-sensitive endpoints. The check should use the lowercased string only for the protocol comparison; the return value should be the original URL: `return lower.startsWith('http://') || lower.startsWith('https://') ? url : null;`.
- **[MEDIUM]** Backend CI tests — @illume's most recent CHANGES_REQUESTED notes backend test failures. Confirm `cd backend && go test ./...` passes after addressing the above.

### Suggestions
- **[LOW]** `charts/headlamp/templates/deployment.yaml` — External links injected via `HEADLAMP_CONFIG_EXTERNAL_LINKS` env var (Helm inline), but the backend flag is `--external-links` parsed from a `koanf:"external-links"` key. Confirm `koanf` maps the `HEADLAMP_CONFIG_EXTERNAL_LINKS` env var correctly (koanf env provider typically uses `HEADLAMP_CONFIG_` prefix with `_` → `-` transformation, giving key `external-links` ✓). Add a brief Helm test or note for this mapping.

### Positive
- **[Positive]** `backend/cmd/headlamp.go:2229` — `externalLinks := []ExternalLink{}` (not nil) ensures the JSON response always contains an array, never `null`, which is safe for frontend consumers.
- **[Positive]** `charts/headlamp/values.schema.json` — `externalLinks` added to the JSON schema with correct type; validates operator-supplied values before deployment.
