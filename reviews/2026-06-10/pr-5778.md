## Summary

The 2026-06-09 commit (`frontend: sidebar: Fix external links sanitization and translation`) resolves the prior `safeLinkUrl` bug (old implementation returned `url.toLowerCase()` which would lowercase case-sensitive URL paths). The new implementation correctly lowercases only the scheme portion. An i18n key for `"External Links"` is also added. However, two persistent issues block merge: the first commit still uses Conventional Commits format (`feat:`) instead of the required headlamp `area: SubArea: Description` style, and the PR body still contains the unresolved placeholder `Fixes #<issue-number>`.

## Findings

### Warnings
- **[HIGH]** `git log` — First commit subject `feat: add configurable external links to sidebar via Helm values` uses Conventional Commits style (`feat:`) instead of the project's required format (`area: SubArea: Description`, e.g. `frontend: Sidebar: Add configurable external links via Helm`). The verb must start with a capital letter. This has been flagged in every prior review — please rebase and amend.
- **[MEDIUM]** PR body — `Fixes #<issue-number>` is still an unresolved placeholder. Either replace with the real issue number or remove the line. This has persisted across multiple re-reviews.
- **[MEDIUM]** `charts/headlamp/values.schema.json` — `externalLinks` is not listed in the schema's `config.properties`. While `additionalProperties` is not set to `false` for the `config` section (so `helm lint` will not hard-fail), adding an array schema entry would enable IDE autocomplete, document the field, and provide validation for consumers of the chart.

### Suggestions
- **[MEDIUM]** `frontend/src/components/Sidebar/useSidebarItems.tsx` — `safeLinkUrl` normalizes the scheme to lowercase but returns the original trimmed URL for the path/query parts. This is correct. Consider adding a brief comment explaining the intent (only the scheme is normalized, not the full URL) to prevent future authors from accidentally re-introducing the lowercase regression.

### Positive
- The `safeLinkUrl` fix correctly limits lowercasing to the scheme (`https?://`) only, preserving case-sensitive path components.
- Backend: `getConfig` initialises `externalLinks` to `[]{}` (not nil) before JSON encoding — the frontend always receives an array, never `null`.
