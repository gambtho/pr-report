## Verdict: APPROVE

## Summary
Updates Cluster Inventory access provider examples to the v0.1.2 images, adjusts binary paths to the new `/bin/` subdirectory layout, adds `interactiveMode: Never`, and introduces `kubeconfig-secretreader` as a second provider. The Helm template validation is improved from an exact directory match to a prefix match (`hasPrefix`), allowing binaries nested inside mount paths. Documentation, test fixtures, and schema are all updated consistently.

## Findings

### Suggestions
- **[LOW]** `charts/headlamp/values.schema.json:299` — `"pattern": "^\/"` escapes the forward slash unnecessarily; JSON does not require escaping `/`. `"pattern": "^/"` is equivalent and cleaner (the original used `^/`). Not a bug, just a minor style nit.

### Positive
- Helm template path validation upgraded from `dir $cmd == mountPath` (only matched a binary at the root of the mount) to `hasPrefix mountPath/ $cmd` — correctly allows binaries in subdirectories like `/bin/secretreader-plugin` under `/access-plugins/secretreader`.
- `interactiveMode: Never` added to exec credential configs — this is the recommended value for non-interactive access providers and was missing from the v0.1.1 examples.
