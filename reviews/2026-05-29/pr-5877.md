## Verdict: APPROVE

## Summary
Clean extraction of `incrementRequestCounter` from `headlamp.go` into the `auth` package as `IncrementRequestCounter`, with an added nil check on `metrics.RequestCounter` that the original lacked. Two well-structured tests use the OTel SDK `ManualReader` to inspect emitted metrics in-process. Single commit with correct format.

## Findings

### Positive
- Added `metrics.RequestCounter == nil` guard on top of the existing `metrics != nil` check — the original `headlamp.go` method only checked the outer struct, which would panic if `RequestCounter` was nil within a non-nil `Metrics`.
- `TestIncrementRequestCounter` uses `sdkmetric.NewManualReader()` to collect and assert the emitted `http.server.request_count` data point with correct attribute values — proper production-code metric testing without mocking.
- `TestIncrementRequestCounter_NoMetrics` verifies both `nil` metrics and `&telemetry.Metrics{}` (nil counter inside) do not panic — covers the defensive guards exhaustively.
