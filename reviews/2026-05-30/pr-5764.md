## Verdict: REQUEST_CHANGES

## Summary

This PR continues to carry two structural bugs introduced in a previous iteration: a duplicate `clusterInventory:` key in `values.yaml` and an extra `}` in `values.schema.json` that makes the Helm JSON schema invalid. The broader feature work (OIDC auto-login flag, frontend redirect logic, configSlice tests, `inventorymetadata` type promotion) is well-structured and functionally correct, but the two schema-breaking bugs and the unresolved commit hygiene issues require another round of fixes before merge.

## Findings

### Critical

- **[HIGH]** `charts/headlamp/values.yaml:134` ŌĆö Duplicate YAML key: `clusterInventory:` appears twice in succession (the `+` line adds a bare `clusterInventory:` immediately before the existing `clusterInventory:` block). YAML parsers discard one silently; Helm will use only the last key, dropping all values in the first block. **Must be removed.**

- **[HIGH]** `charts/headlamp/values.schema.json` (clusterInventory section, end) ŌĆö Extra closing brace `}` is added after the `clusterInventory` section close, producing invalid JSON. Helm will reject the chart with a JSON parse error. **Must be removed.**

### Warnings

- **[HIGH]** `backend/pkg/kubeconfig/kubeconfig.go:1173` ŌĆö CACert regression: the previous code set `CACert: nil` when `oidcCACert == ""`, now it always sets `CACert: &oidcCACert` (pointer to an empty string). Any downstream consumer that checks `if certPath == nil` to determine whether a CA cert was configured will now incorrectly see a non-nil pointer pointing to an empty string. This is a behavior change that should be verified against all callers of `OidcConfig.CACert`.

- **[MEDIUM]** `feature/oidc-auto-login` branch ŌĆö Still contains a `Merge branch 'main' into feature/oidc-auto-login` merge commit. This should be rebased away before merge per project conventions.

- **[MEDIUM]** Commit messages ŌĆö Two commits remain with the generic subject `frontend/helm: Address Copilot review comments` and `frontend/backend: Address Copilot review comments`. These violate the project convention of meaningful commit subjects and make the history unreadable. Each should be squashed into the commit it amends or rewritten with a specific description.

### Suggestions

- **[MEDIUM]** `frontend/src/components/authchooser/index.tsx` ŌĆö The `useEffect` that drives the OIDC auto-login redirect still has no Vitest component test. A test covering (a) redirect fires when `oidcAutoLogin=true` and cluster is OIDC, (b) redirect is suppressed when `sessionStorage` has the attempt flag, and (c) flag is cleared on successful auth would prevent regressions. This has been requested in multiple previous re-reviews.

- **[LOW]** `frontend/src/components/authchooser/index.tsx:91` ŌĆö `oidcUrl.searchParams.set('dt', Date())` uses `Date()` (returns a human-readable string like "Sat May 30 2026 07:20:58 GMT+0000"). A compact integer such as `Date.now().toString()` is the conventional cache-buster and produces a shorter, URL-safe value.

- **[LOW]** `backend/pkg/clusterinventory/clusterinventory.go` ŌĆö All exported-function and internal-helper godoc comments have been stripped in this PR (e.g., `// reconcileRoots computes the desired discovery rootsŌĆ”`, `// contextFromClusterProfile builds a Headlamp contextŌĆ”`, etc.). Go convention requires godoc on exported symbols; removing internal comments reduces future maintainability. Please restore them.

### Positive

- Promoting `inventorymetadata` types (`Metadata`, `Profile`, `Version`, `Property`) into the `kubeconfig` package eliminates a circular-adjacent sub-package and simplifies the import graph. `ClusterInventoryMetadata.DeepCopy()` is a nice addition.
- `configSlice.ts` now has three targeted unit tests covering `oidcAutoLogin` set/preserve/clear semantics. These correctly validate the `'oidcAutoLogin' in action.payload` guard.
- `window.opener` detection in `oidcauth/index.tsx` correctly differentiates popup vs. full-page OIDC callback flows and handles each appropriately.
- `sessionStorage` usage for loop prevention (vs. `localStorage`) is correct ŌĆö session-scoped storage clears on tab close, so the auto-login fires once per session rather than being permanently suppressed.
- `ThemeProviderNexti18n.tsx` fallback to `'en'` when `i18n.language` is falsy is a safe defensive fix for the race condition described in the PR.
