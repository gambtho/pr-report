## Verdict: REQUEST_CHANGES

## Summary

This PR adds `--oidc-auto-login` to automatically redirect users to the OIDC provider when no session is active, and in this revision also wires several Cluster Inventory improvements (removing the separate `inventorymetadata` sub-package in favour of types in `kubeconfig`, de-escalating "experimental/alpha" flags to GA language). The core OIDC auto-login logic (sessionStorage loop-prevention, configSlice protection, OIDCAuth redirect vs popup disambiguation) is now functionally solid. However, three persistent structural defects remain: an **invalid JSON schema** (extra `}` in `values.schema.json`) that will break Helm schema validation on every install, a **duplicate YAML key** in `values.yaml` that silently drops the `clusterInventory` section in strict parsers, and two **merge-main commits** that need to be removed by rebasing. Commit message quality issues also persist across multiple commits.

## Findings

### Critical

- **[HIGH]** `charts/headlamp/values.schema.json:325` — An extra `}` added in this diff creates an unbalanced JSON object. Running `python3 -m json.tool values.schema.json` reports a parse error. Helm schema validation (`helm install --dry-run`) will fail for any chart user. This bug was flagged in the 2026-05-28, 2026-05-29, and 2026-05-30 re-reviews and remains unresolved. Remove the extra `}` on the line immediately after the `clusterInventory` TTL override field.

### Warnings

- **[HIGH]** `charts/headlamp/values.yaml:132` — The diff adds `+  clusterInventory:` immediately before the existing `  clusterInventory:` block, creating a duplicate YAML key. YAML 1.2 (and most Helm parsers) treat duplicate keys as the last-wins, silently losing the bare key entry; strict parsers (e.g. `yq --unwrap-scalar=false`) will error. This was flagged in 2026-05-28, 2026-05-29, and 2026-05-30 re-reviews. Remove the bare duplicate `  clusterInventory:` line added by this PR.

- **[HIGH]** `(commit history)` — Two merge-main commits (`37c37067 Merge branch 'main' into feature/oidc-auto-login` and `f65051b8 Merge branch 'main' into feature/oidc-auto-login`) are still present. Per project convention, `@illume` consistently requests rebases before merge. Please `git rebase main` and force-push to remove merge commits.

- **[MEDIUM]** `(commit history)` — Commits `1d1bb7fa frontend/helm: Address Copilot review comments` and `da9888f1 frontend/backend: Address Copilot review comments` use vague subjects that describe the review process, not the change. Per project conventions, commit subjects must describe what changed (e.g. `frontend: AuthChooser: Clear auto-login flag on sign-in`). These have been flagged in every re-review since 2026-05-27.

- **[MEDIUM]** `(commit history)` — Commit `deecfffb backend: wire Cluster Inventory configuration` is missing the SubArea segment. Should be `backend: clusterInventory: Wire configuration` or similar. Commit `a8f8d966 cluster-inventory: show ClusterProfile health conditions` uses a non-standard area prefix (`cluster-inventory` is not an established area; should be `backend: clusterInventory:`) and has a lowercase verb (`show` → `Show`).

### Suggestions

- **[MEDIUM]** `backend/pkg/kubeconfig/kubeconfig.go:1135` — `CACert: &oidcCACert` passes a pointer to an empty string instead of `nil` when no CA cert is configured (old code used `var caCert *string; if oidcCACert != "" { caCert = &oidcCACert }`). All current consumers (`auth.ConfigureTLSContext`) guard with `caCert != nil && *caCert != ""` so there is no observable functional regression, but the nil-pointer-as-absence convention is broken. Restore the nil guard to keep the field semantics idiomatic and safe for future callers.

- **[LOW]** `backend/pkg/clusterinventory/clusterinventory.go:1` — The package-level godoc comment (`// Package clusterinventory discovers ClusterProfile resources and adds them to Headlamp's context store as Cluster Inventory contexts.`) and the field-level comments on `Options` were removed. These helped orient newcomers and should be retained.

- **[LOW]** `charts/headlamp/values.schema.json:299` — `"pattern": "^\/",` escapes the forward slash unnecessarily (JSON does not require it; `"^/"` is equivalent). Minor, but signals a diff-artifact rather than an intentional change.

### Positive

Well-reasoned approach.
- **[Positive]** `frontend/src/components/oidcauth/index.tsx` — The OIDCAuth component now cleanly handles both the popup flow (signal via `localStorage`) and the full-page redirect flow (clear sessionStorage flag + `history.replace` to cluster page). The disambiguation using `window.opener` is correct.
- **[Positive]** `frontend/src/redux/configSlice.test.ts` — Three targeted tests (`setConfig with oidcAutoLogin`, `preserve when omitted`, `allow clearing`) give precise coverage of the payload guard logic.
- **[Positive]** `backend/cmd/headlamp_test.go` — `TestGetConfigIncludesOidcAutoLogin` validates end-to-end serialisation of the new field through `getConfig`, catching future wiring regressions.
