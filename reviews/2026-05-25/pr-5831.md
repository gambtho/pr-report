## Summary

Adds 11 unit tests for `frontend/src/helpers/recentClusters.ts`, covering `setRecentCluster` (empty storage, string vs Cluster-object, dedup, cap at 3) and `getRecentClusters` (empty, stored list, non-array, null, corrupted JSON). Module had zero prior test coverage; this significantly improves regression safety.

## Findings

### Suggestions
- **[LOW]** `frontend/src/helpers/recentClusters.test.ts:11` — The test hardcodes `const STORAGE_KEY = 'recent_clusters'`; if the implementation ever changes the key, the test's `read()` helper would break at the assertion, which is acceptable as a failure signal. However, consider adding a brief comment noting this is the expected storage key so the contract is explicit.

### Positive
- Tests cover both string and `Cluster`-object signatures for `setRecentCluster`, exercising the polymorphic input the implementation supports — no gaps in the happy path.
- The "throws when stored payload is not valid JSON" test explicitly documents current behavior (SyntaxError surfaces rather than silently returning `[]`), which is good: it will catch any future defensive error-handling change that alters the contract.
