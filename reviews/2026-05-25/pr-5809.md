## Verdict: APPROVE

## Summary

This PR adds milli-bytes (`m`) suffix handling to `parseUnitsOfBytes` in `units.ts`, fixing wildly incorrect memory percentage display in the Node detail view. The SHA changed since the last review; the diff now also supports decimal milli-byte values (e.g. `1.5m`). The implementation is correct, well-tested, and the PR body description is accurate.

## Findings

### Positive
- Regex `/^\d+(?:\.\d+)?m$/` is well-scoped: it matches only plain integer or decimal values ending in `m`, avoiding any conflict with the existing SI-prefix parser (`[BKMGTPEe]` are uppercase, so `m` was unhandled before this change).
- Four test cases cover integer, large integer, large exact, and decimal milli-byte values — good edge-case coverage.
